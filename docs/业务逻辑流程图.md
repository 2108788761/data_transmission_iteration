# 业务逻辑流程图（Mermaid）

以下流程图可用 VS Code 的 Mermaid 插件、Typora 或 [Mermaid 在线编辑器](https://mermaid.live/) 预览；导出为 PNG/SVG 后可插入到文档或汇报中。

---

## 图1：程序整体流程（从启动到上传）

```mermaid
flowchart TB
    Start([启动 Run.py]) --> Login[显示登录窗口]
    Login --> Input[输入账号/密码/油田]
    Input --> Auth{POST /api/v2/token/}
    Auth -->|成功| JWT[获得 JWT]
    Auth -->|失败| Input
    JWT --> Main[打开主窗口 OpenExeWindow]
    Main --> Tab1[Tab1 井信息设置]
    Tab1 --> UploadWell[点击：上传井信息]
    UploadWell --> API_Well[POST crew_wellinfo_state_get]
    API_Well --> Code{状态码}
    Code -->|1002| Allow[允许数据操作]
    Code -->|1006| TipEnd[提示井段已结束]
    Allow --> Choice{传输方式}
    Choice -->|软件直传| Tab2[Tab2 选择文件+映射字段]
    Choice -->|串口转传| Tab3[Tab3 串口接收+上传]
    Tab2 --> StartUpload1[开始上传]
    Tab3 --> Recv[开始接收→写入本地DB]
    Recv --> StartUpload2[开始上传]
    StartUpload1 --> WS1[WorkerThread WebSocket]
    StartUpload2 --> WS1
    WS1 --> Server[(后端服务器)]
```

## 图3：井信息与状态码

```mermaid
flowchart TD
    A[用户选择：区块/井台/井号/压裂次数/层位/段/压裂队/仪表车] --> B[点击上传井信息]
```

## 图4：秒点数据流（直传 + 转传）

```mermaid
flowchart TB
    subgraph 直传
        F1[现场 DB 文件] --> F2[获取字段+映射]
        F2 --> F3[WorkerThread]
        F3 --> WS[WebSocket]

    subgraph 转传
        S1[串口数据] --> S2[ReceiverThread]
```

## 图5：粘密与供水数据流

```mermaid
flowchart TB
    subgraph 粘密
        V1[粘密串口 Modbus RTU] --> V2[ReceiverViscosityThread]
        V2 --> V3[LogData 或 ViscosityData]
        V3 --> V4[UploadViscosityThread]
        V4 --> WSV[WebSocket Viscosity]
    end
    subgraph 供水
        W1[Modbus TCP IP:Port] --> W2[ReceiverWaterThread]
        W2 --> W3[WaterData]
        W3 --> W4[UploadWaterThread]
        W4 --> WSW[WebSocket Water]
    end
    WSV --> Back[(后端)]
    WSW --> Back
```

## 图6：上传线程与断点续传

```mermaid
flowchart TD
    Start[开始上传] --> Build[temp_db_build]
    Build --> Run[启动上传线程]
    Run --> Connect[WebSocket 连接]
    Connect --> Loop[循环]
    Loop --> Check{point_1 < point_2?}
    Check -->|否| ReadDB[查询最新 point_2]
    ReadDB --> Loop
    Check -->|是| Read[读取 point_1~point_2]
    Read --> Send[发送 JSON]
    Send --> Resp{服务端 flag}
    Resp -->|1002| Move[point_1=point_2, 保存 config]
    Resp -->|1004| Retry[回退指针, 重连]
    Move --> Loop
    Retry --> Connect
```
